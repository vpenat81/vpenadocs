    Confidentiality

    Protect the information from unauthorized access and disclosure (Could be an individual, entity or process).

    Keeping information secret and safe from prying eyes, only access who they need to access.

    2. Maintain a business advantage
    3. Achieve regulatory compliance (PII, PHI, Various types of financial data

    How to maintain the Confidentiality

    1. Encryption
    2. Access Controls
    3. Data Masking
    4. Physical Security Measures
    5. Training and awareness

    üì∂ Availability Overview

    Ensures that information, systems, and resources are accessible and operational when needed by authorized users. | Employees can access internal systems 24/7 without downtime. |

    üîé Why Availability Matters

    Benefit Description Example
    Business Continuity Keeps operations running during failures A backup server activates instantly after the main one fails.
    Customer Trust Maintains reliable services Users trust online banking due to constant service uptime.
    Organization's Reputation Avoids negative impact due to outages A retail site remains online during high-traffic holiday sales.

    üîÅ Redundancy Strategies:

    To overcome the challenges associated with maintaining availability, the best strategy is to use redundancy in the systems and network designs.

    Redundancy Type Description Example
    **Redundancy ** Duplication of critical components or functions to enhance reliability. Having a backup internet connection and server for critical services.
    Server Redundancy Uses multiple servers in a load-balanced or failover configuration to maintain service availability. If one server crashes, others continue handling user traffic.
    Data Redundancy Stores data in multiple locations to prevent loss or access failure. Cloud storage syncing data across different regions.
    Network Redundancy Uses multiple network paths to ensure data delivery if one route fails. Internet reroutes through a backup ISP line if the main one is down.
    Power Redundancy Uses backup power sources like UPS or generators to avoid power-related downtime. Data center switches to generator during a city-wide blackout.

    üîêAuthentication

    Verifies the identity of individuals or entities in a communication or transaction

    üîê 5 Authentication Methods Overview

    Authentication Method Description Example
    Authentication (General) Ensures individuals or entities are who they claim to be during a communication or transaction. Logging into a secure website with a password.
    Something You Know (Knowledge) Relies on information that a user can recall. Passwords, PINs, or security questions.
    Something You Have (Possession) Relies on presenting a physical item to authenticate. Security token, smart card, or a mobile authenticator app.
    Something You Are (Inherence) Uses unique physical or behavioral traits of a person. Fingerprint, face scan, or retina scan.
    Something You Do (Action) Depends on a user‚Äôs unique behavior or action. Typing rhythm, voice recognition.
    Somewhere You Are (Location) Uses the user's geographic location as a factor in authentication. Logging in only when located at the office or within a country.

    üîê Zero Trust: Never trust, always verify...

    This mean we can not just trust to the network perimeter to protect the system as a main line of defense, with deperimeterization (common in cloud and remote work and adoption of new technologies)we are put a lot or risk into the organization that is why we need to focus on: strong authentication, data encryption access controls per user/devices, zero trust architecture.

    Zero Trust demands verify for every device, user and transaction within the network, regardless of its origin (it doesn't mater where it comes from: inside or outside the network.

    Types Objective Description
    Control Plane is the brain (making decisions) Adaptive Identity, threat scope reduction, policy-driven access control, secured zones with Policy Engine and Policy Administrator.
    Data Plane is the hands (doing the work) Subject/System and Policy Enforcement Point

    üìå Control Plane

    Control Plane Description
    Control Plane responsible for defining, managing, and enforcing the policies related to user and system access.
    Adaptive Identity validation into account user behavior, device, location and more.
    Threat scope reduction Limit he access only for the need work tasks, avoid risk and compromise credentials.
    Policy-driven access control Enforce user access policies based on their roles and responsibilities.
    Secured zones Isolated environments within a network that are designed to house sensitive data.

    Control Plane uses: Policy Engine and Policy Administrator to makes security decisions.

    Decision-making and enforcement What it does How it works Example Question?
    Policy Engine Makes decisions based on policies Evaluates context like user identity, device health, location, and request type. Decides if a user with a healthy laptop and MFA can access a sensitive app. Should this request be allowed?
    Policy Administrator Enforces the decision from the Policy Engine. Connects to the Policy Enforcement Point (PEP) to allow or block traffic. Sends an ‚Äúallow‚Äù command to the firewall or gateway to permit access. Apply the Policy Engine‚Äôs decision

    The Policy Enforcement Point (PEP) is the security gateway or system that executes the decision: allow, block, or limit user/device access.

    üîÅ Flow Summary:
    1. User makes a request.
    2. Policy Engine evaluates based on policies
    3. Policy Administrator applies the decision via the Policy Enforcement Point (PEP).

    üìå Data Plane

    Component What it Does How it Works Example
    Subject/System Verify the authenticity of the subject or system. The user, device, or application requesting access to resources. Sends access requests and presents credentials or context to the system. An employee‚Äôs laptop trying to access a corporate app.
    Policy Enforcement Point (PEP) Enforces access control decisions by allowing or blocking data flow. Acts as a gateway or proxy that checks with the Control Plane before permitting access. Firewall, secure proxy, or cloud access gateway controlling traffic to an app.

    ‚ö†Ô∏è Gap Analysis

    Is the process used to compare the current state of a process, system, or performance against a desired future state. It helps identify the "gaps" or differences that need to be addressed to achieve the goals.

    Example:
    A company wants to improve its cybersecurity posture. The current state shows basic firewall protection, but the desired state includes advanced threat detection and response capabilities.
    The gap analysis reveals missing tools and processes that must be implemented to reach the desired security level.

    Current State Desired State Identified Gap
    Basic firewall protection Advanced threat detection & response Lack of security monitoring tools
    Manual patch management Automated patch deployment Need for automation in updates
    Limited employee training Regular security awareness training Insufficient user education
