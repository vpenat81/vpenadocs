    üíæ Basic boot process components

    Component Description Example
    BIOS (Basic Input/Output System) Legacy firmware that initializes hardware and starts the bootloader Accessed by pressing F2 or Del during startup
    UEFI (Unified Extensible Firmware Interface) Modern firmware replacing BIOS, with more features and faster booting Accessed via F12 or Esc on newer systems
    mkinitrd Creates an initial RAM disk image for booting mkinitrd /boot/initrd.img-$(uname -r)
    grub2-install Installs the GRUB2 bootloader to a device grub2-install /dev/sda
    grub2-mkconfig Generates a new GRUB2 configuration file grub2-mkconfig -o /boot/grub2/grub.cfg
    grub2-update Updates GRUB2 with the latest kernel (alias for grub2-mkconfig on some systems) grub2-update
    dracut Tool to generate initramfs (initial RAM filesystem) dracut -f
    initrd.img Initial RAM disk used during early boot /boot/initrd.img-5.15.0
    vmlinuz Compressed Linux kernel executable /boot/vmlinuz-5.15.0
    GRUB2 (Grand Unified Bootloader v2) Bootloader that loads and transfers control to the Linux kernel /boot/grub2/grub.cfg
    PXE (Preboot Execution Environment) Boots system from a network server Booting diskless workstations using PXE server
    USB Boot Boots system from a USB flash drive Installing Linux from /dev/sdb1
    ISO Boot Boots system from an ISO image Booting from ubuntu.iso using virtual machine or USB

    üíª Storage concepts

    Concept Description Example
    File Storage Stores data as a hierarchy of files and folders NFS, SMB/CIFS
    Block Storage Stores data in fixed-size blocks for performance and flexibility iSCSI, SAN
    Object Storage Stores data as objects with metadata and unique IDs Amazon S3, OpenStack Swift
    MBR (Master Boot Record) Legacy partitioning scheme, supports up to 4 primary partitions /dev/sda1 using MBR
    GPT (GUID Partition Table) Modern partitioning scheme, supports large disks and more partitions /dev/sda1 using GPT
    FUSE (Filesystem in Userspace) Allows non-privileged users to create and mount their own filesystems sshfs, encfs
    RAID - Striping (RAID 0) Splits data across multiple disks for speed, no redundancy 2 disks striped for performance
    RAID - Mirroring (RAID 1) Duplicates data across two or more disks for redundancy 2 mirrored drives for high availability
    RAID - Parity (RAID 5/6) Uses parity information for fault tolerance with efficient storage 3+ disks with 1 disk of space used for parity
